# Markdown

Markdown es un lenguaje de marcado ligero que se utiliza para formatear texto de manera sencilla y legible. Es ampliamente utilizado en la creación de documentos, especialmente en entornos de desarrollo de software. Markdown permite agregar formato, como encabezados, listas, enlaces y más, sin la necesidad de utilizar códigos complicados. Su simplicidad lo hace ideal para la creación de documentos claros y bien estructurados.

## Opciones de etiquetado de Markdown

Markdown es un lenguaje de marcado ligero que permite agregar formato a tus documentos de texto plano. A continuación, se muestran algunas de las opciones de etiquetado que ofrece Markdown:

- **Encabezados:** Puedes crear encabezados utilizando el símbolo `#`. Por ejemplo, `# Encabezado 1` crea un encabezado de nivel 1.
- **Texto en negrita y cursiva:** Puedes hacer que el texto aparezca en negrita utilizando `**` antes y después del texto. Por ejemplo, `**texto en negrita**` aparecerá como **texto en negrita**. Para cursiva, utiliza `*` en lugar de `**`.
- **Listas:** Puedes crear listas utilizando `-` o `*`. Por ejemplo, `- elemento 1` crea un elemento de lista con viñeta.

## Comandos de Git

Git es un sistema de control de versiones distribuido que te permite realizar un seguimiento de los cambios en tu código fuente. A continuación, se muestran algunos de los comandos más comunes utilizados en Git:

- **Verificar el estado de un repositorio local:** Puedes verificar el estado de tu repositorio local utilizando el comando `git status`.
- **Agregar archivos individuales o globalmente:** Puedes agregar archivos individuales utilizando el comando `git add <nombre del archivo>`. Para agregar todos los archivos modificados, utiliza `git add .`.
- **Agregar comentarios a la confirmación:** Puedes agregar comentarios a la confirmación utilizando el comando `git commit -m "comentario"`.
- **Cargar tus cambios en el repositorio remoto:** Puedes cargar tus cambios en el repositorio remoto utilizando el comando `git push`.
- **Crear, explorar y eliminar ramas:** Puedes crear una nueva rama utilizando el comando `git branch <nombre de la rama>`. Para explorar una rama, utiliza `git checkout <nombre de la rama>`. Para eliminar una rama, utiliza `git branch -d <nombre de la rama>`.
- **Revertir un repositorio a una confirmación específica:** Puedes revertir un repositorio a una confirmación específica utilizando el comando `git revert <hash de confirmación>`.
